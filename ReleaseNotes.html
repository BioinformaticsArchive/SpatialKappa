Summary of additions for v2.0.0

%link: declarations are now %channel: declarations.

Channel and compartments names are now ids rather than labels, so no need to use single quotes.

%init:, %obs:, %var: declarations and transform rules can now specify location. Agents can also specify location.

Agent links can specify channel to define multi-voxel complexes.

An issues and bug tracker will appear eventually. For now, please email any bug reports or feature requests to spatialkappa@demonsoft.org.

Known issues

Validation of invalid models not catching all errors cleanly - requires some extension.

Transform rules limited to single merged complex (on left or right).

Issue handling the transform rule A() -> B() @ 1.0

Location and channel extensions apply only to translator application, not the spatial kappa simulator.

Evaluation of channels for link evaluation travels in forward direction only.

Multiple definitions of same link not handled in translator - only one definition used.
